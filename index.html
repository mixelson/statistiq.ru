Мы помогаем легко, быстро и удобно анализировать огромный массив данных о сервисных транзакциях. Для провайдеров телефонной связи - массив записей о звонках. Для провайдеров платежных сервисов - массив записей о платежах.

Ежедневное и ежеминутное использование сервиса приводит к формированию массива записей. Например, если 100 клиентов каждую минуту совершают вызов, то за три года у вас сформируется как минимум 157 миллионов записей. Это уже не мало для офисного приложения и не много для серверного. Наше решение позволляет с комфортной скоростью работать с миллиардами записей.

Еще один момент, который важен для наших пользователей - скорость обновления данных. Реальность промышленной эксплуатации сервисных систем массового обслуживания зачастую состоит в том, что бизнес-пользователи получают информацию с задержкой в месяц. Соответственно, пропадает возможность оперативно принимать решения об эффективности рекламных кампаний и характере поведения пользователей. STATISTIQ решает и эту боль. Вы получаете данные оперативно - сутки, минуты, секунды - в зависимости от готовности инфраструктуры вашего предприятия.

Для наилучшего пользовательского эффекта мы используем комбинацию баз-данных и стек технологий для работы программного комплекса STATISTIQ.

Простейшим вариантом развертывания может быть использование одного сервера в вашей корпоративной сети. Пользователи получают доступ через Web интерфейс к серверным компонентам Front-End сервера, выполняющим функции авторизации, разграничения прав доступа и защиты данных от несанционированного доступа. Компоненты Front-End взаимодействуют с Beck-End компонентами систем управления базами данных и выполняют логические операции формирования пользовательских наборов данных для визуализации и анализа.

Данные о пользовательских транзакциях поступают из инфраструктуры через систему загрузчиков файлов или потоком через информационную шину. Может быть реализован сбор данных о транзакциях на сетевом уровне - через обработку копии сетевого трафика.
